Get Started

This section aims to provide you with assets to quickly get the Grommet framework running. You'll find recommendations for your application's structure, development, testing, static code analysis, and deployment.

Grommet is based on ReactJS which provides great features in JavaScript for building user interfaces. You'll also use a JavaScript syntax extension called JSX. To enable front-end routing you'll use React Router.

We expect that you have at least a basic understanding on these technologies to be able to master Grommet, in addition to JavaScript of course. In terms of cascading style sheets (css), Grommet provides everything you need to quickly create applications based on the application Style Guide. Under the hood, you'll find InuitCSS and Sass to compile the style sheets. But, don't worry, you are not expected to write a lot of CSS when using Grommet. We've done that for you. But if you would like to contribute, please do so!

Grommet is in the early stages of development. As of this moment, Grommet is not deployed in NPM. This means that early evaluators must build Grommet in your local environment. Once Grommet is publicly released, this step will be removed and Grommet will be published in NPM.

Finally, don't forget to Request Access to the source code if you haven't done so. This too is a temporary step that is only required while the repository is private on GitHub.

Building Grommet

To build Grommet you must first install Git and Node.js. Then run the following commands to publish Grommet in your local npm: 


        git clone https://github.com/HewlettPackard/grommet.git
        

        cd grommet
        

        npm link

You can now start a new project and link that to Grommet:


        mkdir sample-app
        

        cd sample-app
        

        npm link grommet

A node_modules folder will be created which contains Grommet in your local environment.
