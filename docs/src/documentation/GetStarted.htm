Get Started

This section aims to provide you with assets to quickly get Grommet framework up-and-running. We will recommend good practices on application structure, development, testing, static code analysis, and deployment.

Grommet is developed on top of ReactJS which provides great features in JavaScript for building user interfaces. We are also using a JavaScript syntax extension called JSX. To enable front-end routing paths we are using React Router.

We expect that you have at least a basic understading on these technologies to be able to master Grommet, in addition to JavaScript of course. In terms of styling (css), Grommet already provides everything you need to quickly create great applications based on our application Style Guide. Under the hood, we are using InuitCSS and Sass to compile our stylesheets. But, don't worry too much here, we are not expecting you to write a lot of CSS code. After all, why reinventing the wheel?

Currently, Grommet is in early stages of development. As of this moment, we are not deployed in NPM. This means that you, as an early evaluator, must build Grommet in your local environment.

Don't forget to Request Access to the source code if you haven't done so.

Building Grommet

To build Grommet is a prerequisite to install Git and Node.js. Then run the following commands to publish Grommet in your local npm: 


        git clone https://github.com/HewlettPackard/grommet.git
        

        cd grommet
        

        npm link

You can now start a new project and link that to Grommet:


        mkdir sample-app
        

        cd sample-app
        

        npm link grommet

A node_modules folder will be created which contains Grommet bits from your local environment.
