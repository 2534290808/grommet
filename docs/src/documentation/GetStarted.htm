Get Started

This section aims to provide you with assets to quickly get the Grommet framework running. You'll find recommendations for your application's structure, development, testing, static code analysis, and deployment.

Grommet is in the early stages of development. As of this moment, Grommet is not deployed in NPM or Bower. This means that early evaluators must build Grommet in their local environment. Once Grommet is publicly released, this step will be removed.

Finally, don't forget to Request Access to the source code if you haven't done so. This is also a temporary step that is only required while the repository is private on GitHub.

Building Grommet

To build Grommet you must first install Git, Node.js, and Gulp. Then run the following commands to publish Grommet in your local npm: 


        {'git clone https://github.com/HewlettPackard/grommet.git\ncd grommet\ngulp dist # This command generates a production version of Grommet.\ncd dist\nnpm link # This command links the production dist to your local NPM.'}

You can now start a new project and link that to Grommet:


        mkdir sample-app
        

        cd sample-app
        

        npm link grommet

A node_modules folder will be created which contains a symbolic link to Grommet from your local environment.
