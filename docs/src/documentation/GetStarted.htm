Get Started

This section aims to provide you with assets to quickly get the Grommet framework running. You'll find recommendations for your application's structure, development, testing, static code analysis, and deployment.

Grommet is in the early stages of development. As such, you must build Grommet in your local environment. Once Grommet is publicly released, you will be able to get Grommet using NPM or Bower.

Finally, don't forget to Request Access to the source code if you haven't done so. This is also a temporary step that is only required while the repository is private on GitHub.

Get Dependencies

  1. Install Git. For Windows, you may like Git for Windows.
  2. Install Node.js

Get Going

  1. Set your proxy environment variables (only required if you're inside a proxied network) 
    
          {'npm config set http-proxy http://host:port'}
    
          {'npm config set https-proxy https://host:port'}
  2. Install Gulp 
    
            {'npm install --global gulp'}
  3. Clone the Grommet repository 
    
            {'git clone https://github.com/HewlettPackard/grommet.git'}
  4. Move to Grommet directory 
    
            {'cd grommet'}
  5. Generate a production version of Grommet in your local directory 
    
            {'gulp dist'}
  6. Move to the distribution directory 
    
            {'cd dist'}
  7. Windows users link to Grommet from an Administrator console 
    
            {'npm link'}
    Linux and OS X users link to Grommet with sudo 
    
            {'sudo npm link'}

You can now start a new project and link that to Grommet:

  1. Move out of the dist folder 
    
          {'cd ../..'}
  2. Create your project directory 
    
          {'mkdir sample-app'}
  3. Move to your new project directory 
    
          {'cd sample-app'}
  4. Link to Grommet in your new project 
    
          {'npm link grommet'}

At this point, a node_modules folder will be created which contains a symbolic link to Grommet from your local environment.
