Tutorial

In this section you will experience a complete, production-ready TodoApp which demonstrates the following features:

  • Login and Logout operations
  • A dashboard with the list of existing tasks
  • Adding, removing, editing, and completing tasks
  • Managing labels and assigning them to tasks
  • Searching for existing tasks and/or labels
  • Filtering tasks by status and/or labels

Initial Structure

Your TodoApp will use Gulp as the task runner, and Flux as the web application architecture. These frameworks are not required for Grommet to work, but we recommend you explore them as they provide many benefits and simplifications for web-based applications.

At this point, make sure you have Grommet available in your local environment by following instructions on the Get Started page.

Create the following folder structure and files locally: 


        {"/todo-app\n  /src\n    /actions\n    /constants\n    /components\n    /scss\n    /stores\n    index.js\n    index.html\n  gulpfile.js\n  package.json\n"}

  • package.json: required project descriptor for NPM. Used to describe project information and dependencies.
  • gulpfile.js: describes the tasks available for the TodoApp that will be executed by Gulp.
  • src/index.js: main project source file that will connect CommonJS modules containing the TodoApp features.
  • src/index.html: main html to load the TodoApp single-page script.
  • src/actions: possible Flux actions that can be executed in this app.
  • src/constants: all constants of the TodoApp.
  • src/components: all the ReactJS components used in this application.
  • src/scss: Sass folder that will contain Grommet stylesheets.
  • src/stores: possible Flux stores that will handle back-end communication.

Running a Dev Server

To execute your code in your browser, you will use a module bundler (e.g. Webpack, Browserify). Grommet provides a set of tasks in Gulp that contains a lot of development tasks available out-of-the-box for you, including a development server based on Webpack.

Add the following code to your gulpfile.js: 


        {"var gulp = require('gulp');\nvar path = require('path');\nvar devGulpTasks = require('grommet/utils/gulp-tasks');\n\nvar opts = {\n  dist: path.resolve(__dirname, 'dist'),\n  copyAssets: [\n    'src/index.html'\n  ],\n  scssAssets: ['src/scss/**/*.scss'],\n  jsAssets: ['src/**/*.js'],\n  mainJs: 'src/index.js',\n  mainScss: 'src/scss/index.scss',\n  webpack: {\n    resolve: {\n      root: [\n        path.resolve(__dirname, 'node_modules')\n      ]\n    }\n  },\n  devServerPort: 8010\n};\n\ndevGulpTasks(gulp, opts);\n"}

Inside todo-app root folder run the following command to start a dev server:


        gulp dev

If you open http://localhost:8010/webpack-dev-server/, you should be able to see the following:

This dev server has been implemented with Hot Module Replacement enabled. This means that any change inside the project will replace the code running in your browser without refreshing your page. Try changing the title and saving the file. The server will detect this change and will update the components in your browser. This works even for your SCSS files. Because this does not refresh your browser, you do not lose your context in your application. This is a tremendous efficiency gain for developers.
